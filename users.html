Target Users

Patients and Visitors, i.e., teens, adults and senior citizens who are busy and consider their time to be valuable. The above mentioned users will be the primary users of the system. Currently, there are no secondary users for the since it is mainly aimed at being used directly by the visitors and patients.

User Requirements

● They should be familiar with using a smartphone.
● Should be comfortable with sharing their location
● The user should be able to view notifications on device and be able to access it in real time
● The users need to be able to have access to internet at all time

User Tasks

● Find Waiting Room
● Join the Waiting Room
● View Waiting Room Current Status
● Move Down in the Waiting Queue
● Leave the Waiting Room
● View approximate wait time
● Text message with details about the appointment
